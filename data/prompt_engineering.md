# Prompt工程

## 什么是Prompt？

Prompt（提示词）是给LLM的输入指令，指导模型生成期望的输出。

好的Prompt设计可以：
- 提高回答质量
- 减少幻觉
- 控制输出格式
- 引导模型推理

## Prompt的核心要素

### 1. Instruction（指令）

明确告诉模型要做什么：

```
❌: "写一首诗"
✅: "写一首关于春天的七言绝句，要求押韵、意境优美"
```

### 2. Context（上下文）

提供背景信息：

```
你是一位有着10年经验的Python工程师...
用户在开发一个电商网站...
```

### 3. Input Data（输入数据）

要处理的内容：

```
下面是一段用户反馈：...
```

### 4. Output Format（输出格式）

指定输出结构：

```
请以JSON格式输出，包含以下字段：
- sentiment: 情感（positive/negative/neutral）
- confidence: 置信度（0-1）
```

## Prompt工程技巧

### 1. Few-shot Learning

提供示例：

```
任务：判断文本情感

示例1:
输入：我今天买的东西质量太差了！
输出：negative

示例2:
输入：服务态度很好，物流也快
输出：positive

请判断以下文本情感：
我昨天退货了，很不满意
```

### 2. Chain-of-Thought (CoT)

引导推理过程：

```
让我们一步步思考：
1. 首先分析问题...
2. 然后考虑解决方案...
3. 最后得出结论...
```

### 3. Role Definition

设定角色：

```
你是一位资深的AI产品经理，擅长将AI技术与真实业务场景结合。
请从产品角度分析以下场景...
```

### 4. Structure Output

结构化输出：

```
请严格按照以下JSON schema输出：
```json
{
  "summary": "一句话总结",
  "keywords": ["关键词1", "关键词2"],
  "action_items": [{"task": "任务", "priority": "高/中/低"}]
}
```
```

## Temperature和Top-p

### Temperature

控制随机性：
- 0：确定性输出，重复性高
- 0.7：平衡（推荐）
- 1.0：创造性高，但可能不稳定

```python
response = chat.Invoke(
    prompt,
    temperature=0.7
)
```

### Top-p (Nucleus Sampling)

选择概率累积达到p的token：
- 较低：安全保守
- 较高：多样化

通常只设置一个，另一个设为默认值。

## System Prompt vs User Prompt

### System Prompt

全局设定，影响所有交互：

```
你是一个有帮助的AI助手。
回答要简洁、专业。
不知道的问题如实说"我不知道"。
```

### User Prompt

当前交互的输入：

```
帮我解释什么是RAG
```

## 常见问题

### 1. 输出太长

限制：
```
请用不超过100字回答
```

### 2. 输出太短

引导：
```
请详细解释，包含具体例子
```

### 3. 格式错误

明确格式要求：
```
重要：请严格按以下格式输出，不要添加任何解释：
[答案]
[理由]
```

### 4. 幻觉

添加约束：
```
如果你不确定，请回答"不确定"，不要编造信息。
请在回答中引用相关来源。
```
